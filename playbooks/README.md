# Playbooks

Playbooks are repeatable workflows.

Use a playbook when the same task needs commands, ordering, checks, or known failure modes.

## Rules
- Keep steps executable and short.
- Include exact commands.
- State expected results only when they guide debugging.
- Link policies instead of restating them.
